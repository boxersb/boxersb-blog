## 문제

Firefox는 역사적으로 한글 입력 시 이벤트 핸들러를 제대로 트리거하지 못하는 호환성 문제가 있었다.

한글을 입력할 시 파이어폭스의 한글 입력기는 글자 조합을 하기 위해 value 세팅과 이벤트 핸들러 호출을 정지시킨다. 이는 특히 자동완성 기능과 폼 유효성 검사에 영향을 미쳤다.

## 해결책: 재귀적 워크어라운드

`setTimeout`을 사용하여 입력 변경을 재귀적으로 체크하는 기법이다:

1. 이전 입력값을 추적하기 위한 가상 변수(`db`)를 생성한다
2. 현재 입력 필드의 값과 저장된 `db` 값을 비교한다
3. 값이 다르면 `db`를 업데이트하고 의도된 이벤트 핸들링 로직을 실행한다
4. 동일하면 처리를 건너뛰고 `setTimeout`을 통해 핸들러를 다시 호출한다

함수 호출을 이벤트 트리거 동작이 아닌 일반 실행으로 처리함으로써, 브라우저가 업데이트된 입력 값에 올바르게 접근할 수 있게 된다.

## 성능 고려사항

비용이 큰 연산은 실제 값 변경이 있을 때만 실행되므로 CPU 오버헤드는 최소한으로 유지된다. 비교만 수행하는 반복은 무시할 수 있는 비용이며, `clearTimeout` 로직을 추가하면 성능을 더욱 최적화할 수 있다.
